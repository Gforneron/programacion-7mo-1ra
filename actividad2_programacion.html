Guía Básica para Usar Git

Git es un sistema de control de versiones distribuido que te permite gestionar y rastrear los cambios en el código fuente de tu proyecto. Aquí te mostramos los pasos esenciales para empezar a usar Git.

  1. Instalación de Git

    Antes de empezar, necesitas instalar Git en tu máquina. Puedes descargarlo desde el sitio oficial de Git.

  2. Configuración Inicial

    Después de instalar Git, debes configurar tu nombre de usuario y correo electrónico:

    • git config --global user.name "Tu Nombre"
    • git config --global user.email "tuemail@example.com"

  3. Crear un Repositorio

    Para crear un nuevo repositorio, navega al directorio de tu proyecto y ejecuta:

    git init

    Esto inicializa un nuevo repositorio de Git en tu carpeta de proyecto.

  4. Añadir Archivos al Repositorio

    Para empezar a gestionar los archivos con Git, primero tienes que añadirlos al repositorio. Usa el comando git add para añadir archivos específicos:

    git add nombre_del_archivo

    Para añadir todos los archivos nuevos y modificados, usa:

    git add .

  5. Realizar un Commit

    Un commit guarda los cambios en el repositorio. Después de añadir los archivos, realiza un commit con un mensaje descriptivo:

    git commit -m "Mensaje descriptivo del commit"

  6. Ver el Estado del Repositorio

    Para ver el estado actual de tu repositorio y los cambios pendientes, usa:

    git status

  7. Ver el Historial de Commits

    Para ver el historial de commits de tu proyecto, usa:

    git log

  8. Clonar un Repositorio

    Si quieres trabajar en un proyecto existente, puedes clonar un repositorio remoto usando:

    git clone https://github.com/usuario/nombre-del-repositorio.git

  9. Trabajar con Ramas

    Las ramas te permiten trabajar en diferentes funcionalidades de forma aislada. Para crear una nueva rama y cambiarte a ella, usa:

    git checkout -b nombre-de-la-rama

    Para cambiar entre ramas, usa:

    git checkout nombre-de-la-rama

  10. Fusionar Ramas

    Cuando terminas de trabajar en una rama, puedes fusionarla con la rama principal (main o master) usando:

    git checkout main

    git merge nombre-de-la-rama

  11. Subir Cambios a un Repositorio Remoto

    Para subir tus cambios a un repositorio remoto (como GitHub), primero asegúrate de haber añadido el repositorio remoto:

    git remote add origin https://github.com/usuario/nombre-del-repositorio.git

    Luego, sube tus cambios:

    git push origin main

  12. Actualizar tu Repositorio Local

    Para obtener los últimos cambios del repositorio remoto y fusionarlos con tu rama local, usa:

    git pull

Conclusión

Git es una herramienta poderosa que puede facilitar enormemente el manejo de tus proyectos de software. Con esta guía básica, ya estás listo para empezar a usar Git y beneficiarte de sus ventajas.

Puedes obtener más información y recursos en GitHub.
